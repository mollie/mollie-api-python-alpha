                                             | For bookkeeping purposes, the settlement includes an overview of transactions included in the settlement. These transactions are grouped into 'period' objects — one for each calendar month.<br/><br/>For example, if a settlement includes funds from 15 April until 4 May, it will include two period objects. One for all transactions processed between 15 April and 30 April, and one for all transactions between 1 May and 4 May.<br/><br/>Period objects are grouped by year, and then by month. So in the above example, the full `periods` collection will look as follows: `{"2024": {"04": {...}, "05": {...}}}`.<br/><br/>Each period object will contain the following fields:<br/>* `revenue` — An array of revenue objects containing the total revenue for each payment method during this period<br/>* `costs` — An array of cost objects, describing the fees withheld for each payment method during this period<br/>* `invoiceId` — The ID of the invoice created for this period, if the invoice has been created yet<br/><br/>Each `revenue` object has the following fields:<br/>* `description` — A description of the revenue subtotal<br/>* `method` — The payment method, if applicable<br/>* `count` — The number of payments<br/>* `amountNet` — The net total of received funds, i.e. excluding VAT<br/>* `amountVat` — The applicable VAT<br/>* `amountGross` — The gross total of received funds, i.e. including VAT<br/><br/>Each `cost` object has the following fields:<br/>* `description` — A description of the cost subtotal<br/>* `method` — The payment method, if applicable<br/>* `count` — The number of fees<br/>* `rate` — The service rates, further divided into `fixed` and `percentage` costs<br/>* `amountNet` — The net total cost, i.e. excluding VAT<br/>* `amountVat` — The applicable VAT<br/>* `amountGross` — The gross total cost, i.e. including VAT<br/><br/>The example response should give a good idea of what this looks like in practise. |
